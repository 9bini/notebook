## 문서 작성 가이드라인

### 글 작성 원칙
- **명확성**: 복잡한 개념을 이해하기 쉽게 설명
- **실무 중심**: 이론과 실제 구현의 연결
- **구조화**: 계층적 정보 구성과 논리적 흐름
- **완결성**: 각 주제별 독립적이면서도 연관성 있는 내용 구성

### 마크다운 스타일 가이드
- **제목 구조**: # (주제) → ## (개념) → ### (세부사항) → #### (구현)
- **코드 블록**: 언어 지정 필수 (```kotlin, ```yaml, ```bash 등)
- **인용구**: 중요한 개념이나 정의에 사용
- **목록**: 계층적 정보는 중첩 목록, 병렬 정보는 단순 목록
- **링크**: 내부 문서 간 교차 참조 적극 활용

### 내용 구성 패턴
```markdown
# 주제명

## 개념 정의
- 핵심 개념 설명
- 등장 배경 및 필요성

## 실무 적용
- 구체적인 구현 방법
- 코드 예시 포함

## 장단점 분석
- 적용 시 고려사항
- 대안 방식과의 비교

## 실제 사례
- 프로덕션 환경 경험
- 문제점 및 해결 방법

## 참고 자료
- 공식 문서 링크
- 추가 학습 자료
```
