# 라이덜리 2차 스프린트 회고 - 팀 리드 첫 경험

*2025년 8월 7일 스프린트 플래닝*

## 스프린트 개요

**목표**: 와이어프레임 기반 핵심 사용자 인증 & 프로필 기능 개발
**기간**: 2차 스프린트 (예상 2주)
**팀 구성**: 팀 리드(나) + Flutter 신입 개발자 1명

## 스프린트 계획 수립 과정

### 기능 범위 정의
와이어프레임을 분석하여 2차 스프린트 범위를 정했다. 사용자 여정을 고려해 3개 핵심 영역으로 분류:

1. **회원가입 & 로그인** - 앱 진입 필수 기능
2. **프로필 관리** - 사용자 개인화
3. **온보딩** - 신규 사용자 가이드

### 세부 개발 항목 도출

#### 회원가입 & 로그인 상세 화면
- 스플래시 화면
- 시작 화면 (기존 로그인 정보 있으면 바로 메인으로)
- 휴대폰 로그인 화면
- 계정 찾기 화면
- 회원가입 플로우
  - 회원가입 안내화면
  - 휴대폰 본인 인증
  - 거점 설정
  - 개인정보 입력
  - 기존 계정 연결 안내

#### 프로필 관리
- 닉네임/프로필 이미지 설정
- 거점 등록
- 팔로우/팔로잉 관리
- 라이딩 경력 입력
- 라이딩 스타일 설정

#### 설정 화면들
- 알림 설정
- 사용자 설정
- 기타 설정들

## 팀 리드 경험 회고

### 팀원 역량 분석
**팀원 배경:**
- Flutter 첫 경험
- 학교 과제 및 단기 프로젝트 경험만 보유
- 장기간 팀 프로젝트 경험 부족

### 업무 배정 전략
**프로필 관리 기능을 팀원에게 할당**

**선택 근거:**
1. **낮은 리스크**: 인증 로직과 독립적이어서 실패해도 핵심 기능에 영향 없음
2. **적정 난이도**: UI 중심 개발로 Flutter 학습하기 적합
3. **명확한 범위**: 요구사항이 비교적 단순하고 명확함

### 의사결정의 어려움
**핵심 딜레마**: 프로젝트 우선순위 vs 팀원 역량 개발

프로젝트 관점에서는 번개 모임이 채팅이 최우선이지만, 팀원의 성장을 위해서는 쉬운 기능부터 시작하는 것이 합리적이라고 판단했다.

## 회고를 마치며

> **"일감 위임은 역시 어려워"**

이 한 줄이 지금 내 심정을 가장 잘 표현한다. 하지만 어려워도 피할 수 없는, 팀 리드로서 반드시 배워야 할 영역이라는 것도 안다.

완벽하지 않더라도, 팀원과 프로젝트 모두를 생각하며 내린 결정이라면 그것만으로도 의미 있다고 생각한다.

**다음 스프린트에서는** 이번 경험을 바탕으로 더 나은 선택을 할 수 있을 것이라 믿는다.
